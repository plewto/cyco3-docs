RCMP MIDI Player Plugin

rcmp is a terminal based MIDI file player available on github at plewto/rcmp. This plugin allows CYCO to control an instance of rcmp via OSC messages. In use, rcmp creates a media-list by scanning a directory for MIDI files. At any one time a single item from the list is selected for playback.

Prior to using the plugin, an rcmp server must be running in a separate terminal. Further the prefix, host-ip and port numbers must match between CYCO and rcmp .

Loading the rcmp plugin binds an instance of RCMP:RCMP-PROXY to RCMP:*PROXY*. The default proxy parameters are:

  • OSC path prefix → /rcmp
  • Server host address → #(127 0 0 1)
  • Server port → 7000

If you need to change any of these values, execute RCMP:RCMP-PROXY

(RCMP:RCMP-PROXY &key prefix host port)

Create new instance of RCMP-PROXY and bind it to RCMP:*PROXY* as the new default for all RCMP functions.

    
    :prefix - OSC address prefix, default /rcmp
    :host   - rcmp server ip address, default #(127 0 0 1)
    :port   - server port number, default 7000
(RCMP:HELP)

Display rcmp help.

(RCMP:SCAN directory)

Add all MIDI files in directory to the media-list. The media-list is cleared prior to scanning the directory. The first file in the directory is automatically selected for playback. NOTE: It can take rcmp several seconds to refresh after scanning a directory.

(RCMP:RESCAN)

Clear the media-list and rescan the current directory.

(RCMP:SELECT item)

Select item from media-list for playback.

    
    The item may be selected either by its numeric list position, or
    by name.
(RCMP:?MEDIA)

Print the media-list

    
    NOTE: The output is produced in the terminal where rcmp is running, not
    in the terminal running CYCO.
(RCMP:PLAY)

Command rcmp to start playback of selected MIDI file.

(RCMP:STOP)

Command rcmp to halt playback.

    
    It may take a second or two for playback to actually stop.
(RCMP:EXIT)

Command rcmp to terminate.
