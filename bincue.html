Binary Cuelist

A Binary Cuelist is a cuelist where event times are indicated by binary values. The primary advantage is that a binary value gives a visual representation of the rhythm.

The BINBALL and BINXBALL macros combine a BINCUE object with QBALL and XBALL parts.

(BINCUE &key symbols timesig use-subbeats)

Returns a BINCUE object which translates a binary-cuelist to the form expected by the BAR cue function.

    
    :symbols - Association list of user symbol expansions.   The BINCUE
               provides several short-hand symbols which may be used in the
               cuelist.  The user may define additional symbols as an
               association list:
    
               :symbols '((clave-1 . “1000 0010 0010 0000”)
                          (clave-2 . “0000 1000 1000 0000”)) 
    
               Use the ?BINCUE method for a list of available symbols. 
                          
    
    :timesig - Time signature or time signature designator.
               The time signature must match the signature for the part(s)
               using the binary cuelist.  The default uses the time signature of 
               the current section of the current project.
    
               Alternatively any time-signature object may be used or a list of the form 
               (bar beat subbeats)
    
               :timesig '(8 6 4) ;; 8-bars 6/4 time with 4 subbeats per beat.
    
    
    :use-subbeats - Boolean, by default the time-signature subbeats (typically sixteenth notes) 
                    are used for the base time unit.  Set this value to nil to use the tsbubeats 
                    (16th note triplets) instead.  
    
                    Tsubbeats may only be used if subbeats is even.
(?BINCUE bincue-object)

Displays available cuelist symbols.

(BINCUE-TRANSLATE bincue-object binary-cuelist)

Performs translation of binary cuelist to BAR form.

    Example
    (param bc (BINCUE :timesig '(2 4 4)))
    (param cuelist (BINCUE-TRANSLATE bc '(1000 0010 0010 0010
                                          1010 0010 0010 0000)))
    
    (print cuelist) → ((1 1 1)(1 2 3)(1 3 3)(1 4 3)(2 1 1)(2 1 3)(2 2 3)(2 3 3))
    
    
    BINCUE-TRANSLATE may also take a string of bits as a cuelist.
    
    (bincue-translate bincue-object "1100 1110 1000 1010")
    White-space is ignored.
