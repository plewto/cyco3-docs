CYCO is a lisp based music composition language with a rich set of pattern generators and a project centric design. For a background on its development see history.

Features

  • Project centric design.
  • Functional time signatures.
  • A rich set of algorithmic pattern generators.
  • Symbolic names for common music parameters.
  • Chord tables.
  • Hierarchical instrument definitions, IE a snare may be a sub-instrument of a drum-machine.
  • Plugins
  • Generates MIDI files for output.

Usage Overview

Briefly, compositions are defined by writing a series of Lisp files in a dedicated project directory. Once defined the project is rendered to a standard MIDI files. An external player is required to play the MIDI files.

Dependencies

The basic CYCO installation has no dependencies other then an ANSI Common Lisp. It has been tested with SBCL, CLISP and ABCL on Linux. Minimal test with SBCL on win32 appears to work. Steel Bank Common Lisp is recommended. Some of the plugins requires quicklisp to install osc, usocket and uiop packages.

Installation

  1. Place the cyco distribution folder in a convenient location. 
    The default location is ~/cyco3
  2. Create the following directories:
       ~/.config/cyco/
   ~/.config/cyco/plugins/
   ~/cyco-projects/
    If you elect to use alternate locations, modify the file cyco/src/local-config.lisp accordingly.

  3. In a terminal navigate to the cyco directory and start Lisp.

  4. At the Lisp prompt enter

    (LOAD "build")

  5. After the source files load, enter the command

    (CYCO)

  6. Run the optional test suite by entering

    (load "test/cyco-test")

    You must be in the cyco directory to run the test. After running the test, restart cyco before preceding.

  7. CYCO is now ready for use. However you may wish to customize it to match your available MIDI equipment.
    You do this by creating plugins in the ~/.config/cyco/plugins/ directory as detailed elsewhere.

  8. If you are using SBCL you have the option of saving the current state of CYCO as an executable.

    (SNAPSHOT executable-name)

    Snapshots are considerably faster then running from source. After starting CYCO from a snapshot you must first enter (CYCO) at the initial Lisp prompt. By default the executable is saved as ~/bin/cyco

Example projects

The examples folder contains several tutorial projects. To use, the contents of examples should be copied to the cyco-projects folder.

  • ex1 - A very simple piano and flute folk song.
  • ex2 - Typical pop-song using Eno's The Great Pretender.
  • ex3 - Charles Ives's The Unanswered Question
  • ex4 - Patterens and Generators
  • ex5 - Strummer part & chords
  • ex6 - Controllers and Bender parts
