BINBALL & BINXBALL

The BINBALL and BINXBALL macros combine a BINCUE object with a QBALL and XBALL respectively. The resulting objects appear as QBALLs or XBALLs in the project tree.

The argument list for both BINBALL and BINXBALL are identical to their respective part types with the following exceptions.

  • There is no :cuefn keyword.
  • The :cue keyword takes a 'binary cue-list' as per BINCUE, or a special form for Euclidean rhythms.
  • There is a new :symbols keyword to define user BINCUE symbols.
  • The is a new :use-subbeats keyword to switch between subbeats and tsubbeats.
  • There is a new :duck keyword for cuelist ducking.

The cuelist may take any of the following forms:

  1. Binary cuelist as per BINCUE.
  2. (EUCLID points shift invert) as with EUCLID.
  3. (EUCLID2 split (p1 s1 i1)(p2 s2 i2)) for use with EUCLID2.
    Where split is the bar number. If nil the half way point is selected.
    (p1 s1 i1) are the point, split and invert arguments respectively for the first rhythm.
    (p2 s2 i2) are arguments for the 2nd rythom.

The :DUCK argument may be:

  1. A BAR format cuelist.
  2. A binary string “1100 1001 ...” cuelist, whitespace is ignored.
  3. Another part object.

    (section a)
    
    (binball a-kick kick
    	 :bars 1
    	 :cue '(1000 1000 1000 1000)
    	 :key 'X
    	 :amp 'ff)
    
    (binball a-hat hat
    	 :bars 1
    	 :duck a-kick
    	 :cue '(euclid 0.75)
    	 :key 'X)
    
    (pprint-cuelist (property a-hat :cuelist))
    ;; → (1 1 3) (1 1 4) (1 2 3) (1 2 4) (1 3 3) (1 3 4) (1 4 3) (1 4 4) 
    ;; Hat never hits on a kick down beat.
(BINBALL name instruments &key ...)

Creates new QBALL using binary-cuelist and bound to name.
