Pattern Comprehension

Pattern Comprehension is a means to express nested pattern structures in a more concise way at the cost of flexibility.

The traditional way to construct deeply nested patterns uses nested list 

    (param p1 (cycle :of (list 'A 'B 'C (dice :of (list 'APE (line :of '(BAT CAT)))))))

Using pattern-comprehension an identical structure is constructed with a single quoted expression. 

    (param p2 (pattern-comprehension '(cycle (A B C (dice (APE (line (BAT CAT))))))))

Two things to note

  1. There is a single quote.
  2. The :OF keyword is not used.

Actually the :OF keyboard may be used if it helps readability, but it is completely optional and has no effect. 

    '(cycle :of (A B C (dice :of (APE (line :of (bat cat))))))

The following generator and pattern types are supported.

  • BAG
  • CYCLE
  • DICE
  • LINE
  • WALKER
  • BONES
  • COUNTER
  • HAILSTONE
  • LOGISTIC
  • RECAMAN
  • RAMP
  • SHIFT-REGISTER

Several of these pattern/generator types take optional keyword arguments. The following keywords may be used where they would normally be expected.

  • :BY
  • :EVEN
  • :FINAL
  • :FUNCTION
  • :HOOK
  • :LENGTH
  • :MASK
  • :ODD
  • :OF
  • :PRERUN

This list of supported pattern types and keywords are reserved words and should be used with caution.
