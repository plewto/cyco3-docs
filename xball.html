XBall

An XBALL combines the features of a QBALL (the use of patterns to recombine parameters) and a STRUMMER (the use of named chords and note strumming). Additionally most XBALL parameters may use pattern comprehension.

Since most XBALL features are detailed elsewhere, only where the XBALL deviates from the QBALL and STRUMMER will be listed here.

(MAKE-XBALL name instrument &key ...)
(XBALL name instrument &key ...)

Creates new instance of XBALL.

    XBALL and MAKE-XBALL are identical except the later binds the new instance
    to the symbol name while the former does not.  The name argument should be
    quoted for MAKE-XBALL and unquoted for XBALL.
    
    Unlike a QBALL, an XBALL may only use single instrument.
    
    Keywords:
    
       :section      - Defaults to current section
       :cuefn        - Defaults to section value.
       :shuffle      - Defaults to section value.
       :shift        - Defaults to 0.0
       :render-once  - Defaults to NIL
       :tempo        - Defaults to section value. 
       :unit         - Defaults to section value.
       :bars         - Defaults to section value.
       :beats        - Defaults to section value.
       :subbeats     - Defaults to section value.
       :transposable - Defaults to t
       :reversible   - Defaults to t
       :chord-model  - Defaults to section value.
       :cue          - As with QBALL
       :key          - As with QBALL except nested list are not allowed.
                       May use pattern comprehension
       :dur          - As with QBALL, may use pattern comprehension.
       :amp          - As with QBALL, may use pattern comprehension.
       :chord        - As with STRUMMER, may take pattern comprehension.
       :inversion    - As with STRUMMER, may take pattern comprehension.
       :octave       - As with STRUMMER, may take pattern comprehension.
       :strum        - As with STRUMMER, may take pattern comprehension.
       :direction    - As with STRUMMER, may take pattern comprehension.
       :end-together - As with STRUMMER.
       :remarks      - Optional remarks text
       :reset-on-repeat - As with QBALL.

See BINXBALL for a version of XBALL which uses binary cuelist.
