CUELIST and PPRINT-CUELIST

(CUELIST object &key form timesig use-subbeats)

Returns cuelist from object.

    
    object   - May be any of the following:
                 1. A cuelist in standard BAR form.
                 2. A string of binary digits "1100 1010 1000 1101",  white space is ignored.
                 2. A PART object.
    
    :form    - Result format.  By default CUELIST returns a list in BAR format.
               If :form is :binary, the result is a binary-string.
    
    :timesig - Reference time-signature, may be one of:
                 1. Any instance of time-signature, such as a project, section or part.
                 2. A list of form '(bars beats subbeats) to use a local time-signature.
                 3. NIL.  The default is to use the current-section of *project*.
                    If there is no current-section, then *project* itself is used.
                    If *project* is nil, a fallback 2-bar signature in 4/4 time is used.
    
    :use-subbeats - Boolean, if true use time-signatures subbeats (typically four 16th notes per beat)
                    as the shortest time interval.   Otherwise use tsubbeats (typically six 16th note 
                    triplets per beat).  The default is t for subbeats.
    
    It is important that the time-signature matches the expected signature of the object argument. 
    They do not have to be completely the same. At a minimum the bar-count, beats-per-bar, and 
    subbeats-per-beat must match, otherwise you will get unexpected results.
(PPRINT-CUELIST object &key header form timesig use-subbeats)

Pretty prints cuelist.

    
    With exception of the additional :header keyword, arguments to PPRINT-CUELIST are 
    identical to CUELIST.
    
    :header - Prints optional header text
