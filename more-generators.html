More Generators

Hailstone

(HAILSTONE seed &key even odd hook monitor action)

Returns generator which produces a hailstone sequence A061641

    
    seed  - Integer, starting value.
    :even - Function applied to even result, default (lambda (n)(/ 2 n))
    :odd  - Function applied to odd result, default (lambda (n)(1+ (* 3 n)))
    :hook - Function applied to output value, default (lambda (n) n)
    :monitor - Predicate function called within NEXT-1.  When monitor returns
               non-nil the action function is executed.
               Default (lambda (value) nil)
    :action  - Function called within NEXT-1 whenever monitor is non-nil.
               The result becomes the next internal value.
               Default (lambda (hailstone value) value)

Bones

(BONES &key function hook monitor action)

Random number generator

    :function - Random number generator, defaults to (lambda ()(random 1.0))
                The PDF plugin defines several suitable functions.
    :hook     - Value hook, defaults to (lambda (n) n)
    :monitor - Predicate function called within NEXT-1.  When monitor returns
               non-nil the action function is executed.
               Default (lambda (value) nil)
    :action  - Function called within NEXT-1 whenever monitor is non-nil.
               The return value is not used.
               Default (lambda ())

Logistic

Image credit

(LOGISTIC &key prerun seed mu hook monitor action)

Returns generator which uses a Logistic map.

    
    :prerun - Integer, number of time to step generator after it is created or reset, default 0.
    :seed   - Float, initial value, 0.0 < seed ≤ 1.0, default 0.5
    :mu     - Float, 'fecundity' parameter, 0.0 ≤ mu ≤ 4.0, default 3.25
    :hook   - Value hook function, default (lambda (n) n)
    :monitor - Predicate function called within NEXT-1.  When monitor returns
               non-nil the action function is executed.
               Default (lambda (value) nil)
    :action  - Function called within NEXT-1 whenever monitor is non-nil.
               The return value becomes the next logistic value.
               Default (lambda (logistic value) value)

Euclid

(EUCLID length points &key shift hook monitor action)

Returns Euclidean rhythm generator. See on-line demo

    
    Euclid divides a given integer length as evenly as possible by the number
    of points.  If points does not evenly divide into length, the generated
    points have unequal spacing.  This turns out to be particularly useful for
    poly-rhythms
  
    
       (param ec (euclid 8 5))
       (next ec 5) → (0 2 3 5 6)
    
       The length argument of 8 determines the gamut of possible values 0..7.
       These may be considered the 8 8th notes in a 4/4 bar.   The points
       argument 5 selects specific notes out of this set.  
    
       To be used as a rhythm the 'raw' Euclid output must be converted to a
       cue-list.  The generator hook function is ideal for this conversion.
    
       (param ec (euclid 8 5
                          :hook #'(lambda (n)
                                     (let ((sub (if (evenp n) 1 3))
                                           (beat (truncate (+ (* n 1/2) 1))))
                                       (list 1 beat sub)))))
    
        (next ec 5) → ((1 1 1) (1 2 1) (1 2 3) (1 3 3) (1 4 1))
    
    
        The shift argument produces the same pattern but shifts the selected notes
        
        (param ec (euclid 8 5 :shift 2))
        (next ec 5)  → (0 2 4 5 7)
    
    
    
        Typical Euclid implementations require the following conditions:
    
             Length and points are positive integers where points ≤ length.
    
        This implementation allows points > length, this will result in
        duplicate values.  
    
        (param ec (euclid 8 9))
        (next ec 9)  → (0 1 2 3 4 4 5 6 7)
